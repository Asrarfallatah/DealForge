# Abstract

DealForge is a web-based platform designed to simplify the process of discovering, managing, and tracking business deals through a centralized digital workspace. The platform enables users to organize opportunities, monitor deal progress, manage related information, and streamline communication within a secure and intuitive environment.

The system focuses on providing a scalable architecture, responsive user experience, and secure backend services while following modern software engineering practices.

---

# Project Objective

Traditional deal management often relies on spreadsheets, emails, and disconnected tools, resulting in:

- Limited visibility into deal progress
- Inefficient organization of deal information
- Difficult collaboration between stakeholders
- Time-consuming manual tracking
- Lack of centralized data management

DealForge addresses these challenges by providing a unified platform that improves organization, productivity, and overall workflow efficiency.

# System Design

## 1. Database ERD

The database consists of the following core entities:

- User
- Deal
- Category
- Client
- Status
- Transaction
- Activity
- Notification
- Attachment
- Role

---

## 2. System Architecture

The project follows a layered architecture:

- Controller Layer
- Service Layer
- Repository Layer
- Entity Layer
- Security Layer
- Database Layer

---

## 3. Project Flow

The application workflow is as follows:

1. User Registration / Login
2. Authentication & Authorization
3. Deal Creation
4. Deal Management
5. Progress Tracking
6. Status Updates
7. Completion / Archive
